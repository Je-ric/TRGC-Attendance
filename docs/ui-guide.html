UI / UX Design Guide

Kingdom Ledger · TRGC Attendance System · v1.0 · Feb 2026

Design Principle. Kingdom Ledger is a data-first tool used during or after church services, often on mobile. The UI should feel purposeful and efficient — trust, warmth, and clarity over decoration. The TRGC brand (maroon + gold) anchors visual identity; everything else serves the user task.

06 — Guidelines
Do's & Don'ts

Quick reference rules for maintaining visual consistency.

✓ Borders over shadows Use a 1px #EAE3D0 border to define cards. Rely on shadows only for elevation (modals, floating elements).
✗ No heavy shadows Avoid box-shadow: 0 8px 30px ... on cards or buttons. It makes the UI feel heavy and dated.
✓ Consistent radius Cards always 10px. Inputs/buttons 6px. Badges 4px. Modals 12px. Nothing above 12px.
✗ No excessive rounding Avoid rounded-full on anything other than avatars and circular icon buttons.
✓ One primary action per page The gold-gradient button should appear once per screen context — the most important action.
✗ No competing primaries Never put two gold/primary buttons side-by-side. One primary, the rest ghost or maroon.
✓ Use modals outside .kingdom-shell Always render modals via @push('modals') or @teleport('body') to avoid backdrop-filter clipping.
✗ No nested fixed elements Never place position:fixed inside a container with backdrop-filter or transform.
✓ Data-dense but airy Use enough white space between sections (40px+), but keep form controls compact (9px padding). Data tables should show 8–10 rows without scrolling on desktop.
✗ No decorative patterns on data screens The maroon + gold background radial gradients are subtle. Don't add textures, patterns, or decorative images on data-heavy pages.
✓ Mobile-first interaction Check-in happens on mobile during a live service. Touch targets minimum 44×44px. Checkboxes full-row interactive.
✗ No tiny tap targets Icon-only buttons under 32px in data tables are hard to tap during service under pressure.
CSS Classes to Implement. Replace the current rounded-2xl (16px) on .kingdom-shell with rounded-xl (12px). Replace rounded-xl (12px) on cards with rounded-[10px]. Replace rounded-xl on inputs with rounded-[6px]. Keep .ui-card, .ui-card-soft, .ui-btn class names — just adjust the radius and shadow values inside them.
